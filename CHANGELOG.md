# Changelog

All notable changes to this skill will be documented in this file. The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1.0] — 2026-05-12

### Added
- **Step 0: Establish research question(s) or objective(s) first.** A new mandatory step before Phase 1 that elicits the research question upfront and saves it to `step0_research_questions.md`. The question is then explicitly referenced in Phases 3, 4, 5 and 7 so coding stays disciplined and themes are judged on relevance to the question, not just frequency.
- **Writing register: strictly academic.** A new section in Phase 7 that enforces a formal academic voice for the manuscript body, suitable for peer-reviewed publication. Specifies third-person, hedged claims, active voice for analyst actions, tense conventions, integrated citations, disciplined sentence structure and a ban on metadiscourse padding.

### Changed
- Phase 1 no longer re-elicits the research question. It carries forward from Step 0.
- Phase 3 opens with a reminder to re-read the saved research question(s) before coding starts.
- Personal writing-style skills are now explicitly overridden for the manuscript body. They still apply to ancillary outputs (plain-language summaries, blog versions of the findings) when requested separately.

## [1.0.0] — Initial release

### Added
- Full Braun and Clarke (2006) six-phase framework wrapped as a skill workflow.
- Four upfront analytic decisions made explicit (rich description vs detailed account, inductive vs theoretical, semantic vs latent, epistemological stance).
- 15-point quality checklist and five common pitfalls bundled as reference files.
- Manuscript-style Word document output via the docx skill.
- Annotated thematic map output as PNG.
- APA 7th Edition referencing via the apa-referencing skill.
